Configure

配置

sugar.config.js即我们的配置，主要分为4部分。

  1. template --> 模板相关的配置。
  2. build --> build相关的配置。
  3. server --> 开发服务器相关的配置。
  4. extra --> 额外的配置。

以下是默认配置（sugar.config.js可以选择覆盖其部分或全部）：

const { resolve } = require('path')

module.exports = {
    template: {
        root: __dirname,
        // pattern to list groups
        groups: '{group,set}/*',
        templateExt: '.html',
        dataExts: ['.yml', '.yaml', '.json', '.js'],
        configFilename: 'project',

        shared: 'common',
        helper: 'helpers',
        data: 'data',
        view: '',
        partial: 'components',
        layout: 'layouts',
        defaultPage: 'index',
        defaultLayout: 'index'
    },
    build: {
        // Array of patterns, used to include/exclude html pages,
        // such as `[**/*.html, !myTmpProj/**.html]`
        htmls: [],
        dest: resolve('dest'),
        ignoreBuiltinPatterns: false,
        assets: true
    },
    server: {
        host: '0.0.0.0',
        port: 3000
    },
    extra: {
        disableCache: true,
        mergeAssets: true
    }
}

每项的功能如下：

配置项 功能
template 模板相关的配置
template.root 模板的根目录，用于查找共享目录/页面位置，required
template.groups pattern，列出group目录
template.templateExt 指定页面的后缀，默认".html"
template.dataExts 指定数据文件的后缀，默认['.yml', '.yaml', '.json', '.js']
template.configFilename 指定子项目配置文件名字，该配置可以覆盖我们这里的配置的template
template.shared 指定共享目录，默认"common"
template.helper 指定自定义helper的所在目录，默认'helpers'
template.data 指定数据文件的所在目录，默认'data'
template.view 指定页面的所在目录，默认''
template.partial 指定partial的所在目录，默认'components'
template.layout 指定layout的所在目录，默认'layouts'
template.defaultPage 指定默认的页面名，默认'index'
template.defaultLayout 指定默认的layout名，默认'index'
build build相关的配置
build.dest dest目录
build.htmls pattern，用于筛选页面
build.ignoreBuiltinPatterns 是否禁用内置的筛选pattern
build.assets 是否复制静态资源
server 开发服务器相关的配置
server.host host，默认0.0.0.0
server.port port，默认3000
extra 开发服务器相关的配置
extra.disableCache 是否禁用缓存（partial/data等等）
extra.mergeAssets 是否合并静态资源（指通过css/jshelper引用的样式和脚本）
