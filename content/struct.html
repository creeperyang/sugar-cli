Directory Structure

目录结构

sugar-cli使用简单而灵活的目录结构，既可以适应若干页面的简单项目，也能适应有多个子项目的复杂项目。这种灵活性通过配置来达到。

基本目录结构如下:

.
├── common
│   ├── assets
│   │   └── normalize.css
│   ├── components
│   │   ├── foot.html
│   │   ├── head.html
│   │   └── header
│   │       ├── header.less
│   │       └── index.html
│   └── layouts
│       └── default.html
├── demo
│   ├── assets
│   │   └── style
│   │       └── index.scss
│   └── index.html
└── sugar.config.js

相应说明：

文件 说明
sugar.config.js 配置文件（required）
common 共享文件所在目录，可在sugar.config.js中定义
common/components 放置共享的partial
common/layouts 放置共享的layout
common/assets partial/layout用到的静态资源，包括样式/脚本/图片等等
demo 可视作子项目，或者单纯的目录（组织多个页面）
demo/index.html 页面
demo/assets 页面用到的静态资源，包括样式/脚本/图片等等
