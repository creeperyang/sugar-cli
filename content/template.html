Template

Template

sugar的内置模板是sugar-template，其语法基于mustache而增加了一些定制：

  • {{ variable }} 表达式，输出转义过的值。</li>
  • {{{ varialbe }}} 表达式，输出未转义过的值。</li>
  • {{& varialbe }} 等同上面。</li>
  • {{= raw text =}} 直接输出原文。</li>
  • {{! varialbe }} 注释，输出空字符串。</li>
  • {{#helper}} code {{/helper}} helper，功能由helper自身决定。</li>
  • {{ helper data }} inline helper。</li>
  • {{ data | filter1 | filter2 }} data经若干filters处理后输出值。</li>

一个简单的例子：

{{#if user}}
  {{> "./components/user.html" user}}
{{else}}
  <p>no user.</p>
{{/if}}

Helpers

helper是一个函数，接收参数（包括包裹的block和else block）并返回处理后的字符串。

sugar-template内置了一些helpers：if/unless/each。通过这些可以做基本的输出控制。

InlineHelpers

inline helper即inline的helper。

sugar-cli提供了css/js/url等强大的inline helper，智能地处理静态资源和其它。

{{css url}} inline helper

{{css "./one.scss"}} // --> ''
{{css "./two.scss" mergeAssets=false}} // --> <link>

css helper会读取并编译url指定的样式（scss/less/postcss），

  • 如果mergeAssets为falsy，则返回<link href=correctUrl>，并保存编译后的样式到这个correctUrl；
  • 否则，返回空字符串。
  • 最终合并所有mergeAssets为truthy的样式，保存到一个correctUrl，并插入<link href=correctUrl>到<head>标签底部。

{{js url}} inline helper

基本和css helper类似，除了最终合并到脚本是插入到<body>标签底部。
